A specialisation of ISO 19110 constraint, adding an association with a property, because Metafor constraints are conditions on the value of one property that results in a particular set of possible values for another property.<br /></p><p><br /></p><p>The inherited constrainedBy association is used to represent the property that is constrained by the constraint (that is, the property that is only available to the user if the constraint's condition is fulfilled), which itself has associations to the possible values for that property.<br /></p><p><br /></p><p>The conditionProperty and conditionValue associations are used to represent the details of the condition, together with the constraintType attribute.<br /></p><p><br /></p><p>For example, for the constraint:<br /></p><p>if ReferenceDate is not "present-day" then make available to the user a property BathymetryAdjustment with values coastal only, full adjustment or none.<br /></p><p>conditionProperty = ReferenceDate<br /></p><p>conditionValue = present-day<br /></p><p>constraintType = IS NOT<br /></p><p>constrainedBy = ReferenceDate.<br /></p>
