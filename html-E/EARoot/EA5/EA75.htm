A process describes a specific real world process.<br /></p><p><br /></p><p>A process may have sub components:<br /></p><p>e.g. atmosphere, and<br /></p><p>ocean, and there is at the process level a one-to-one mapping between the scientific and software descriptions.<br /></p><p><br /></p><p>These mappings go down the hierarchy until such time as the underlying mappings are no longer one to one, in which case the composition relationships are independently described within the scientific description and the software description component hierarchies.<br /></p><p><br /></p><p>In practice then, a process must have a scientific description, if it doesn't then any detail is handled in the software description.<br /></p><p><br /></p><p>Other possible names<br /></p><p>component, software, process, physicalprocessorfeatureofatmosphere,scheme etc.<br /></p>
